# 智能续写功能使用指南

## 概述

智能续写系统是小说AI生成与管理的核心功能，它能够基于前文内容、项目设定和人物世界观配置，智能生成连贯的章节内容。

## 功能特点

### 1. AI智能续写
- 基于前文内容智能生成后续章节
- 保持人物性格和故事风格的一致性
- 支持首章生成和章节续写两种模式

### 2. 上下文分析
- AI自动分析情节发展需求
- 推荐相关的角色、世界观设定和情节节点
- 提供智能化的创作建议

### 3. 多版本生成
- 一次生成2-3个不同风格的版本
- 版本之间通过temperature参数调整变化程度
- 方便作者选择最适合的写作风格

### 4. 实时进度反馈
- WebSocket实时推送生成进度
- 显示当前生成版本和进度百分比
- 支持中断和错误提示

### 5. 三种控制模式
- **简单模式**：使用默认参数快速生成
- **标准模式**：自定义基础参数（字数、温度等）
- **高级模式**：详细控制生成参数和上下文

## 使用流程

### 1. 准备工作

#### 创建项目和基础设定
1. 登录系统后创建新项目
2. 填写项目基本信息（标题、类型、风格、简介）
3. 添加主要角色人物：
   - 主角/反派/配角的人物设定
   - 性格特点和核心动机
   - 样貌和身份背景
4. 配置世界观设定：
   - 时代背景和地域特色
   - 核心规则和力量体系
   - 文化和历史背景

#### 创建章节大纲
1. 在项目章节管理中创建新章节
2. 编写章节大纲和摘要
3. 设置章节字数目标

### 2. 开始生成

#### 方法一：智能生成（推荐）
1. 进入项目详情页面
2. 点击"智能生成章节"按钮
3. 选择生成模式：
   - **首章模式**：生成第一个章节
   - **续写模式**：基于前文生成后续章节
4. 配置生成参数：
   - 生成版本数量（1-3个）
   - 字数目标
   - 温度参数（影响创造性）
   - 生成模式（简单/标准/高级）

#### 方法二：基于上下文的智能生成
1. 点击"分析上下文"按钮
2. AI自动分析并推荐：
   - 相关人物角色
   - 世界观设定
   - 情节节点
3. 接受或调整AI推荐
4. 开始生成

### 3. 查看生成进度

1. 生成开始后会显示进度窗口
2. 实时查看生成状态：
   - 开始生成提示
   - 当前版本进度
   - 完成百分比
3. WebSocket连接确保实时更新

### 4. 选择和编辑版本

1. 生成完成后显示所有版本
2. 查看每个版本的：
   - 内容预览
   - 字数统计
   - AI生成的摘要
3. 选择最佳版本：
   - 点击"选择此版本"
   - 可以在应用前编辑内容
   - 选中的版本会应用到章节中

### 5. 后续处理

1. 应用选中版本到章节
2. 查看完整章节内容
3. 可以继续编辑或：
   - 标记为"修订中"
   - 继续续写下一章
   - 导出章节内容

## 最佳实践

### 1. 项目设定准备
- **详细的角色设定**：越详细的角色设定生成的内容越准确
- **完整的世界观**：包含时代、文化、规则等核心要素
- **清晰的项目风格**：确定小说的整体风格和语调

### 2. 生成参数配置
- **字数建议**：一般章节2000-3000字为宜
- **温度设置**：
  - 0.7-0.8：保守，接近已有风格
  - 0.8-0.9：平衡，有适当创新
  - 0.9-1.0：创造性强，风格变化大
- **版本数量**：建议生成2个版本，兼顾效率和选择

### 3. 内容优化
- **选择性使用**：取多个版本的优点组合
- **人工润色**：AI生成后需要人工调整语言和细节
- **保持一致性**：注意检查与前文的衔接和人物一致性

### 4. 定期维护
- **清理旧版本**：系统会自动清理7天前的未选中草稿
- **更新设定**：根据创作进展调整人物和世界观设定
- **保存重要版本**：重要内容建议手动备份

## 常见问题

### Q: 生成的内容不符合预期怎么办？
A:
1. 检查项目设定是否详细准确
2. 尝试调整temperature参数
3. 使用高级模式手动选择上下文
4. 分段生成，先生成大纲再填充内容

### Q: 如何保证内容连贯性？
A:
1. 确保前文内容完整
2. 保持人物设定的一致性
3. 定期使用上下文分析功能
4. 逐步生成，避免跨度太大

### Q: 能否生成特定风格的文本？
A:
1. 在项目风格中设置目标风格
2. 使用高级模式调整提示词
3. 可以在生成前提供风格示例
4. 通过temperature控制创造性程度

### Q: 多个版本如何选择？
A:
1. 对比不同版本的情节走向
2. 检查人物行为的合理性
3. 选择语言风格更符合的版本
4. 可以组合多个版本的优点

## 技术说明

### 生成流程
1. **上下文收集**：收集项目、角色、世界观设定
2. **上下文分析**：AI分析前文和情节需求
3. **推荐生成**：推荐相关实体和生成策略
4. **内容生成**：生成多个版本的草稿
5. **保存草稿**：保存到content_generation_drafts表
6. **版本选择**：用户选择后应用到正式章节

### 数据结构
- **草稿表**：保存所有生成版本
- **版本管理**：支持版本比较和回滚
- **上下文缓存**：优化重复生成性能

### 性能优化
- 异步生成避免阻塞
- WebSocket实时反馈
- 草稿自动清理机制
- 列表分页加载优化
