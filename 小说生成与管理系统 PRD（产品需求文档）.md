# 二、核心需求概述（用户视角）

## 2.1 核心需求

1. 多小说管理：支持同时创建、编辑、归档多本小说，数据隔离，可快速切换项目

2. 维度化设定：可精准配置小说的世界观、人物、情节、主题等核心要素，支持关联与复用

3. AI辅助创作：基于设定自动生成大纲、章节、对话等内容，支持风格定制与人工修订

4. 逻辑一致性管控：系统自动检查人设、时间线、规则的矛盾点，提醒伏笔回收与剧情节奏

5. 版本与进度管理：记录创作历史，支持版本回溯、分支剧情推演，直观查看创作进度

6. 多格式输出：支持将创作内容导出为常用格式，满足投稿、发布需求

## 2.2 非核心需求（后续迭代）

1. 协作功能：支持多用户共同编辑同一小说，分配编辑权限

2. 素材市场：提供付费/免费的背景、人物、情节模板，供用户下载复用

3. 发布对接：直接对接小说平台（如起点、晋江），支持一键发布章节

4. 数据分析：生成创作行为报告，优化创作效率与内容质量

## 2.3 需求优先级（MoSCoW 法则）

|优先级|定义|核心需求示例|
|---|---|---|
|Must have（必须实现）|核心功能，缺失则产品无法使用|多小说创建/管理、基础设定配置、简单AI生成、章节编辑|
|Should have（应该实现）|提升用户体验，核心场景必需，可后续迭代补充|版本管理、实体关联、基础逻辑校验、单格式导出|
|Could have（可以实现）|锦上添花，不影响核心功能，视资源情况迭代|多格式导出、灵感黑板、数据可视化|
|Won’t have（暂不实现）|非当前阶段重点，后续视需求反馈调整|协作功能、素材市场、平台对接|
